# AI 影视工作室 UI 重构设计文档

- 日期：2026-03-29
- 主题：AI 影视制作产品 UI 重构
- 适用范围：
  - `/workspace`
  - `/workspace/[projectId]`
  - `/workspace/asset-hub`
- 目标用户：独立创作者
- 优先目标：提升品牌与专业感

## 1. 设计结论

本次 UI 重构不定义为“界面美化”，而定义为一次面向业务链路的产品体验重构。

产品应被定义为 `AI 影视工作室操作系统`，而不是若干 AI 能力页面的集合。整套重构采用以下总策略：

- 结构策略：`分镜中枢型`
- 视觉策略：`混合工作室系统`
- 体验策略：`克制、精密、稳定的商业动效`

核心判断：

- `storyboard` 是产品中心舞台
- `videos` 是结果验证层
- `editor` 是深色沉浸精修层
- `assets` 是跨项目支撑层

## 2. 页面定位

### 2.1 项目列表页 `/workspace`

定位：`工作室总控台`

职责：

- 展示全局生产状态
- 让用户快速查看任务进度、生成结果、异常项
- 让用户低成本回到最值得继续处理的项目

不应再被定义为：

- 项目陈列墙
- 以“新建项目”为中心的入口页
- 纯管理后台列表

### 2.2 项目详情页 `/workspace/[projectId]`

定位：`单项目生产现场`

职责：

- 承载项目的核心生产链路
- 以分镜为中心组织创作、确认、生成、回看
- 串联资产、视频、配音、编辑器等下游能力

不应再被定义为：

- 多阶段平均排布的标签页容器
- 功能堆叠式后台页面

### 2.3 资产中心 `/workspace/asset-hub`

定位：`跨项目资产母库`

职责：

- 沉淀角色、场景、声音等高复用资产
- 提供可视化检索、编辑、引用、回流项目能力
- 为项目生产提供一致性与效率支持

不应再被定义为：

- 单纯素材仓库
- 图片墙式资源页

## 3. 业务链路分析

### 3.1 主业务链路

建议定义为：

`进入系统 -> 查看结果与进度 -> 回到当前最值得处理的项目 -> 围绕分镜推进生产 -> 进入视频结果验证 -> 进入编辑器精修 -> 沉淀资产用于后续复用`

### 3.2 三个页面在链路中的关系

- 项目列表页：负责 `总控与回流`
- 项目详情页：负责 `主生产`
- 资产中心：负责 `长期沉淀`

三者不是并列关系，而是前后衔接关系。

### 3.3 用户核心目标

对独立创作者而言，来到系统后的首要目标不是新建项目，而是：

- 快速知道哪些任务完成了
- 快速知道哪里失败了
- 快速进入最该继续处理的项目和阶段

进入项目后的核心目标不是浏览所有功能，而是：

- 找到当前最值得推进的分镜组或镜头
- 完成必要确认
- 推进到视频生成和结果回看

### 3.4 关键业务优先级

建议优先级定义为：

`结果总览 > 回到生产现场 > 分镜决策 > 结果验证 > 精修输出 > 资产沉淀`

而不是：

`项目管理 > 功能导航 > 资产维护 > 编辑器`

## 4. 页面结构优化建议

## 4.1 项目列表页 `/workspace`

建议结构：

1. 顶部总览带
2. 继续工作区
3. 项目列表区
4. 新建项目入口

### 顶部总览带

显示：

- 进行中项目数
- 待处理异常数
- 最近完成结果数

目的：

- 让用户一进系统先理解生产状态

### 继续工作区

这是首屏核心区域，优先展示：

- 最近活跃项目
- 最近完成的生成结果
- 失败任务
- 待继续阶段

目的：

- 告诉用户现在最该点哪里

### 项目列表区

项目卡片或列表应显示：

- 当前阶段
- 最近结果
- 异常状态
- 最近更新时间
- 下一步动作

不应只显示：

- 名称
- 描述
- 创建时间

### 新建项目入口

保留，但降为次主按钮。

原因：

- 用户最高频动作已确认不是新建，而是回看进度和结果

## 4.2 项目详情主工作台 `/workspace/[projectId]`

建议结构：

1. 顶部项目控制带
2. 左侧阶段导航
3. 中央分镜主舞台
4. 右侧上下文面板
5. 底部或次层结果带

### 顶部项目控制带

显示：

- 项目名
- 当前阶段
- 当前剧集
- 总体进度
- 关键状态
- 当前主 CTA

要求：

- 像工作室控制台，不像普通后台页头

### 左侧阶段导航

作用：

- 提供流程骨架
- 告知当前位置和后续路径

要求：

- 不抢主舞台
- 不平均突出所有阶段

### 中央分镜主舞台

这是页面绝对中心，应默认围绕 `storyboard` 组织。

展示重点：

- 当前最值得处理的分镜组
- 镜头状态
- 待确认项
- 可继续动作
- 与角色、场景、生成结果相关的关键上下文

### 右侧上下文面板

按当前阶段动态切换：

- 分镜阶段：角色、场景、提示词、任务状态
- 视频阶段：生成结果、失败信息、重试能力
- 编辑阶段：时间线、输出参数、版本信息

### 底部或次层结果带

展示：

- 最近生成结果
- 失败提示
- 可回看资产

目标：

- 形成即时反馈，但不打断主舞台

## 4.3 资产中心 `/workspace/asset-hub`

建议结构：

1. 顶部资产状态带
2. 左侧分类与文件夹
3. 中央资产网格
4. 右侧详情与动作面板

### 顶部资产状态带

显示：

- 资产总数
- 最近新增
- 最近使用
- 待整理项

### 左侧分类与文件夹

组织逻辑建议：

- 角色、场景、声音为一级结构
- 文件夹为二级组织方式

要求：

- 资产类型优先于文件夹组织

### 中央资产网格

优先强调：

- 预览质量
- 标签
- 最近使用
- 被哪些项目引用

不应只做成图片堆叠展示。

### 右侧详情与动作

展示：

- 资产描述
- 版本
- 引用项目
- 可执行动作

重点动作：

- 用于当前项目
- 设为参考
- 继续编辑

## 5. 模块优先级建议

### P0 主模块

- 当前任务状态
- 当前最值得继续的项目或剧集
- 分镜主舞台
- 最近生成结果
- 明确主 CTA

要求：

- 首屏可见
- 视觉层级最高

### P1 支撑模块

- 阶段导航
- 角色、场景、声音上下文
- 资产引用
- 结果筛选
- 错误与异常提示

要求：

- 离主任务近
- 不与主舞台竞争视觉焦点

### P2 弱化模块

- 新建入口
- 历史记录
- 次要筛选项
- 低频配置
- 冗长说明文案

要求：

- 不抢首屏
- 不抢主按钮

### 结构原则

1. 所有页面都先回答“现在最该做什么”
2. 主按钮永远服务当前阶段目标
3. 信息层级围绕当前生产任务排序，不按功能平均分布
4. 单屏只能有一个主视觉焦点

## 6. 视觉风格定义

本产品的视觉风格定义为：

`专业影视工作室 + 精准数字制作台`

高级感来源：

- 秩序感，不是装饰感
- 材质感，不是炫光感
- 对比控制，不是颜色堆叠
- 局部沉浸，不是全局压暗

### 6.1 风格关键词

- 专业
- 沉浸
- 克制
- 精准
- 流动
- 冷静
- 可信
- 制作感
- 高密度但不压迫
- 有光泽但不过度反射

### 6.2 版式建议

#### 工作台浅色区

适用：

- 项目列表
- 配置
- 总览
- 状态回看

版式：

- 宽屏控制台布局
- 顶部控制带 + 主内容区 + 侧边上下文区

#### 创作深色区

适用：

- storyboard
- videos
- editor

版式：

- 中心舞台布局
- 主画布或主卡片显著放大
- 周边工具退居两侧

#### 过渡区

适用：

- assets
- review
- 结果查看
- 选择器弹层

版式：

- 中性容器
- 作为浅色秩序与深色沉浸之间的桥接层

### 6.3 留白建议

- 页面外边距：`24-32px`
- 模块间距：`16 / 20 / 24px`
- 大模块呼吸区：`32-40px`
- 卡片内边距：`16-20px`

原则：

- 宁可减少模块数量，也不要把页面切成太多小盒子

### 6.4 圆角、阴影、描边、质感建议

#### 圆角

- 顶层容器：`24px`
- 常规卡片：`18-20px`
- 小组件、输入框、按钮：`12-14px`
- 微标签：`999px`

#### 阴影

- 浅色区：轻阴影，更依赖半透明层次和细描边
- 深色区：阴影更弱，主要依赖明暗面对比和边界光

#### 描边

- 浅色区：冷灰蓝低对比描边
- 深色区：半透明蓝灰描边

结论：

- 描边比阴影更重要，决定整体精密感

#### 质感

- 工作台区：轻玻璃拟态，但降低雾度和反光
- 创作区：磨砂金属 + 暗色控制台质感
- 局部允许冷色高光，不允许大片霓虹

### 6.5 字体层级建议

- 主标题：现代几何无衬线，字重 `600-700`
- 模块标题：`600`
- 正文：中性无衬线，字重 `400-500`
- 标签与状态：`500`
- 数字与参数：可使用稳重等宽或半等宽风格

建议字号：

- H1：`32-40`
- H2：`24-28`
- H3：`18-20`
- Body：`14-15`
- Meta：`12-13`

原则：

- 标题收紧
- 正文克制
- 辅助信息保持可读，不故意过灰

## 7. 高级感配色方案

整体色彩策略：

`蓝灰冷中性色 + 低饱和钢蓝主色 + 少量青冰色强调`

不采用：

- 娱乐化高饱和配色
- 紫色科技感
- 大面积金黑影视配色

### 7.1 核心色

- Primary：`#5E8BFF`
- Primary Deep：`#3E63DD`
- Accent Cyan：`#79C9FF`

说明：

- Primary 用于当前操作、选中态、流程焦点
- Primary Deep 用于深色区高优先级激活态
- Accent Cyan 仅用于局部高光和边缘强调

### 7.2 浅色工作台色板

- 页面背景：`#EDF2F7`
- 二级背景：`#E6EDF4`
- 卡片背景：`rgba(255,255,255,0.68)`
- 强卡片背景：`rgba(255,255,255,0.82)`
- 标题文字：`#0F172A`
- 正文文字：`#334155`
- 弱化文字：`#64748B`
- 边框色：`rgba(122,138,158,0.18)`

### 7.3 深色创作区色板

- 页面背景：`#12171D`
- 二级背景：`#181F27`
- 卡片背景：`rgba(21,28,36,0.88)`
- 强卡片背景：`rgba(27,35,45,0.96)`
- 标题文字：`#F3F7FB`
- 正文文字：`#C7D2E0`
- 弱化文字：`#8A97A8`
- 边框色：`rgba(154,170,192,0.14)`

### 7.4 过渡区域策略

- 保持同一套主色、边框、文字对比策略
- 根据系统或模块模式映射为浅或深
- 不额外定义一套独立品牌色系

### 7.5 状态色

- 成功：`#3FAF8E`
- 警告：`#D9A441`
- 错误：`#D96C6C`
- 进行中：`#5E8BFF`
- 已暂停/待处理：`#7B8798`

规则：

- 状态色用于点、线、标签、左边标
- 不用于大面积整块铺色

### 7.6 配色使用规则

1. 单页面只保留一个主强调色焦点
2. 主色负责“当前操作”，状态色负责“当前状态”
3. 浅色区靠透明度和层次感建立高级感
4. 深色区靠材质与边界感建立高级感

## 8. 动效规范建议

动效目标：

1. 帮用户理解状态变化
2. 让工作流切换更顺
3. 提升专业、稳定、精密的系统质感

总原则：

`短、轻、稳、可预期`

### 8.1 动效时长

- 悬停反馈：`120-160ms`
- 按钮点击：`90-140ms`
- 卡片出现：`180-240ms`
- 列表批量出现：单项 `160-220ms`，错峰 `20-40ms`
- Tab 切换：`180-220ms`
- 抽屉与弹窗：`220-280ms`
- 页面首屏进入：`280-420ms`
- 成功反馈：`160-220ms`
- 错误提示：`120-180ms`

### 8.2 缓动曲线

- 默认：`cubic-bezier(0.22, 1, 0.36, 1)`
- 快速反馈：`cubic-bezier(0.2, 0, 0, 1)`
- 退出与收起：`cubic-bezier(0.4, 0, 1, 1)`

要求：

- 避免强弹性 spring

### 8.3 页面进入动效

- 浅色工作台页：`opacity + translateY(8-16px)`
- 深色创作页：`opacity + scale(0.985 -> 1)`
- 导航与工具条可比主内容提前 `40-60ms`

### 8.4 卡片与列表动效

- 轻微上浮 + 透明度变化
- 列表采用短错峰
- 不做夸张飞入

### 8.5 Tab 切换

- 内容区域轻淡入
- 配合 `2-4px` 水平位移
- 顶部激活条平滑滑动
- 不做大幅滑屏

### 8.6 悬停与点击

悬停：

- 卡片边框提亮
- 背景轻微上浮
- 阴影变化极弱

点击：

- 轻微压缩到 `0.985-0.99`
- 颜色收紧一阶

### 8.7 弹窗与抽屉

- 弹窗：`opacity + scale(0.96 -> 1)`
- 抽屉：`opacity + translateX/Y(20px)`
- 遮罩同步淡入
- 不做夸张弹出

### 8.8 加载、成功、错误反馈

加载：

- 页面级：低对比骨架屏
- 模块级：轻骨架或状态条
- 生成中：细进度线 + 状态标签 + 当前步骤说明

成功：

- 用短暂高亮边框、状态条、结果卡片提亮
- 深色区可使用弱蓝青扫光

错误：

- 错误标签 + 原因摘要 + 建议动作 + 重试按钮
- 不做剧烈抖动或整屏红警报

### 8.9 性能原则

1. 优先使用 `opacity` 和 `transform`
2. 避免大面积 blur 动画
3. 避免复杂阴影动画
4. 列表与画布区域优先稳定

## 9. 可直接用于 AI 生成设计稿的提示词

### 9.1 总提示词

设计一个 AI 影视制作工作室产品的桌面端 UI 系统，定位为专业影视工作室操作系统，服务独立创作者，从项目管理到分镜生成、视频生成、编辑输出的一体化工作流。整体风格参考 DaVinci Resolve、Figma、Linear，但不是简单拼贴。界面气质要求专业、沉浸、克制、精准、流动。采用混合工作室系统：工作台和管理区域使用浅色玻璃拟态与冷灰蓝秩序感，创作区域使用深色影视控制室风格，但保持统一设计令牌、圆角体系、边框体系和字体层级。产品中心舞台是 storyboard 分镜工作台，视频区是结果验证层，editor 是深色沉浸精修层，asset hub 是跨项目资产母库。界面要有高级商业信任感，不要娱乐化，不要炫酷霓虹，不要消费级花哨设计。强调清晰层级、宽屏控制台布局、大块面留白、低对比边框、轻玻璃材质、冷静钢蓝主色、青冰色局部高光。

### 9.2 项目列表页提示词

设计 AI 影视制作平台的项目总控台首页，首屏重点不是新建项目，而是查看任务进度、最近生成结果、异常提醒、继续工作入口。浅色工作台主题，玻璃拟态卡片，冷灰蓝配色，顶部是总览控制带，中间是继续工作区和结果回看区，下方是项目列表。整体像专业工作室调度台，而不是普通 SaaS 项目卡片页。

### 9.3 项目详情主工作台提示词

设计 AI 影视制作项目详情工作台，中心舞台是 storyboard 分镜生产控制台。页面包含顶部项目控制带、左侧阶段导航、中央分镜主舞台、右侧上下文信息区、下方结果反馈带。风格为混合工作室系统，浅色结构秩序中嵌入深色创作模块，强调镜头卡片、状态标签、生成结果、继续操作。界面必须体现 AI 影视制作流程，不是通用后台。

### 9.4 资产中心提示词

设计 AI 影视制作产品的资产中心页面，作为跨项目角色、场景、声音的资产母库。页面强调资产复用、引用关系、最近使用、标签系统和快速回到当前项目的操作。风格为中性过渡层，介于浅色工作台和深色创作区之间，要求高级、克制、专业，不要做成普通素材图库。

### 9.5 动效提示词

为专业影视工作室类桌面产品设计克制型商业动效，页面进入轻微淡入和平移，卡片轻上浮，Tab 切换平滑，弹窗轻缩放，加载反馈稳定，不使用夸张弹性，不使用霓虹炫技，强调高级感、流畅感、稳定感、控制感。

## 10. 最终建议

将整套方案压缩为一句话：

`用分镜中枢定义产品灵魂，用混合工作室系统定义视觉语言，用克制精密的商业动效定义体验质感。`
